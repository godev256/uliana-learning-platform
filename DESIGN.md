# Design

## Theme

Light, restrained, editorial. Галерейная сдержанность сохраняется, но страница теперь должна
**продавать**: оффер вынесен в первый экран, доказательства — на видное место, а путь к записи
открыт из каждой секции. Портрет остаётся центральным артефактом, палитра — тёплый холст,
шалфей и глина.

Приоритет — **десктоп**. Мобильная версия сознательно отложена (см. `ДАННЫЕ-ОТ-УЛЬЯНЫ.md`).

Ash, Pebble и Clay затемнены до контраста ≥ 4.5:1 на холсте — заявленный в `PRODUCT.md`
WCAG AA теперь действительно выполняется. Никаких градиентов и многоцветных иконок.

## Typography

Два голоса, строгое разделение ролей:

- **Jost** (класс `.display`) — заголовки, лейблы, кнопки, навигация, цены. Всегда КАПСОМ, с разрядкой.
  Это главный жест бренда.
- **Golos Text** — все бегущие абзацы, обычным регистром, без разрядки. Сделан под кириллицу.
  Капслок в абзаце превращает текст в ряд одинаковых прямоугольников: нет выносных элементов,
  по которым читатель узнаёт слово по силуэту.

Правило: **капс = заголовки и лейблы, обычный регистр = абзацы.**

Шкала: `11 / 12 / 15 / 16 / 18 / 20 / 30 / 34 / 46 px`.
H1 — 46px, H2 — 30px, цена — 34px, статистика в герое — 26px.
Прежняя плоская шкала (всё в 11–18px) не давала иерархии: H1 был на 2px крупнее подписи,
а цена — мельче названия тарифа.

## Layout

Первый экран — полноширинный сплит 50/50: цветной портрет слева, выровненная по левому краю
колонка справа. В ней сверху лого и квалификатор, в середине оффер (надзаголовок → H1 →
подзаголовок → CTA), внизу — строка из трёх доказательств, закрывающая бывшую пустоту.

Секции разделены крупным воздухом. Навигация — внизу страницы, точками (8 ссылок).
Липкой шапки нет: это в анти-референсах.

## Components

- **Hero split** — портрет `<picture>` (AVIF/WebP/JPEG, 640/960/1254), оффер, CTA, строка доказательств.
- **About** — четыре карточки: собственный балл ЕГЭ, образование, опыт, результаты учеников.
- **Method roadmap** — шесть шагов 3×2, включая «обратную связь родителю».
- **Format** — шесть карточек 3×2: онлайн-доска, конспекты, длительность, классы, ДЗ, разбор ошибок.
- **Reviews carousel** — три карточки на десктопе, типографский инициал вместо аватара,
  метаданные (класс, цель) и результат глиняным цветом. Разметка `figure` / `blockquote` / `figcaption`.
- **Pricing** — два тарифа. Цена 34px, у абонемента перечёркнутая старая цена и бейдж «−6%».
  Обе кнопки ведут на запись: оплаты пока нет, и обещать её нельзя.
- **FAQ** — `<details>` на тонкой линии, плюс/минус вместо шеврона.
- **Booking form** — имя, контакт, класс, цель + обязательное согласие на обработку ПДн.
  Поля — тонкая линия без заливки и теней. Отправка: deep-link в Telegram с готовым текстом,
  бэкенд не нужен.
- **Section CTA** — тихая текстовая ссылка в конце метода, формата, отзывов и FAQ.
  Закрывает разрыв в ~3700px между героем и записью.
- **Pill controls** — радиус 50px, 12px капс, sage-dark.

## Motion

Framer Motion: загрузочная хореография героя, появление секций, активное состояние навигации.
Всё деградирует под `prefers-reduced-motion`. При печати раскрытие принудительно снимается,
иначе PDF уходит пустым.

## Подводные камни

- Элементные сбросы обязаны жить в `@layer base`. Бесслойный CSS бьёт утилиты Tailwind независимо
  от специфичности: `button { color: inherit }` вне слоя уже однажды съел `text-bone` и сделал
  стрелку карусели невидимой.
- `content-visibility: auto` на секциях удалён: браузер считал позиции якорей по фейковым высотам
  и промахивался мимо секций на 200–400px.
- `id` полей формы не должны совпадать с `id` секций (`#contact`), иначе якорь ведёт в поле ввода.
- Картинки не импортируем из JS: preload-сканер их не видит. Портрет лежит в `public/`,
  генерируется через `npm run build:images`.
